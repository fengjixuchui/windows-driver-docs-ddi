# VideoPortStallExecution function


## -description

The <b>VideoPortStallExecution</b> function retains control of the processor for the specified number of microseconds and returns to the caller.

## -parameters

### -param Microseconds

Specifies the delay interval, in microseconds.

## -returns

None

## -remarks

Maximum acceptable values for <i>Microseconds</i> are thousands of microseconds during miniport driver initialization. Otherwise, the given delay interval must be no more than 50 microseconds. In general, <b>VideoPortStallExecution</b> can be called only if the miniport driver must wait for a very few microseconds for its adapter to update state.

While a miniport driver's <a href="/windows-hardware/drivers/ddi/video/nc-video-pvideo_hw_interrupt">HwVidInterrupt</a> or <a href="/windows-hardware/drivers/ddi/video/nc-video-pminiport_synchronize_routine">HwVidSynchronizeExecutionCallback</a> function can call <b>VideoPortStallExecution</b>, the miniport driver should be designed to avoid such a call if at all possible. Delays while running at high hardware priorities adversely affect the overall I/O throughput of the system and can freeze the machine.

If a miniport driver has work to be done at regular intervals of more than 50 microseconds, it should implement the <a href="/windows-hardware/drivers/ddi/video/nc-video-pvideo_hw_timer">HwVidTimer</a> function. Calls to a miniport driver-supplied <i>HwVidTimer</i> function at approximately one-second intervals can be enabled with <a href="/windows-hardware/drivers/ddi/video/nf-video-videoportstarttimer">VideoPortStartTimer</a> and disabled with <a href="/windows-hardware/drivers/ddi/video/nf-video-videoportstoptimer">VideoPortStopTimer</a>.
